```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤
│ 1. 완제품 공정명 │ 2. 메인 공정명 │  3. 작업 요소명       │ ← 1단 헤더
├─────────────────────────────────────────────────────────────┤
│ 완제품명+라인    │ 공정NO+공정명  │4M│ 작업요소           │ ← 2단 헤더
├─────────────────────────────────────────────────────────────┤
│ 완제품 제조라인  │ 10 자체입고    │MN│ 00작업자           │ ← 데이터 행
```

## 🔍 현재 폰트 크기

### 1. 페이지 헤더
```typescript
// P-FMEA 구조분석(2단계)
fontSize: 13px
fontWeight: 900
background: #1a237e (진한 파란색)
color: white
```

### 2. 1단 헤더 (메인 타이틀)
```typescript
// "1. 완제품 공정명", "2. 메인 공정명", "3. 작업 요소명"
fontSize: 11px  ⭐
fontWeight: 900
height: 25px
background: #1976d2 (파란), #388e3c (초록), #f57c00 (주황)
color: white
```

### 3. 2단 헤더 (컬럼명)
```typescript
// "완제품명+라인", "공정NO+공정명", "작업요소"
fontSize: 10px  ⭐
fontWeight: 700
height: 22px
background: #90caf9 (연한 파란), #a5d6a7 (연한 초록), #ffcc80 (연한 주황)
color: black
```

### 4. 4M 열 헤더
```typescript
// "4M"
fontSize: 8px
fontWeight: 700
width: 20px
```

### 5. 데이터 셀
```typescript
// 일반 텍스트
fontSize: 10px  ⭐
fontWeight: 400 (일반)

// 공정명 (중요 항목)
fontSize: 10px
fontWeight: 600

// 플레이스홀더
fontSize: 10px
fontWeight: 700
color: #c62828 (빨간색)
```

## ⚠️ 문제점

### 1. 글씨 크기 불일치
```
페이지 헤더: 13px (너무 큼)
1단 헤더:    11px
2단 헤더:    10px
데이터 셀:   10px
4M 열:       8px (너무 작음)
```

### 2. 볼드 남용
```
fontWeight: 900 (페이지 헤더, 1단 헤더)
fontWeight: 700 (2단 헤더, 4M, 플레이스홀더)
fontWeight: 600 (공정명)
fontWeight: 400 (일반 텍스트)
→ 4단계 굵기는 과도함
```

### 3. 높이 불일치
```
1단 헤더: 25px
2단 헤더: 22px
→ 일관성 없음
```

## ✅ 표준화 제안 (사용자 의견 기반)

### 기준
- **완제품공정 레벨 good** → 1단 헤더 크기를 기준으로
- **2단 헤더 = 1단 헤더와 통일**
- **워크시트 영역 = 완제품공정 라인 크기로 통일**

### 제안 A: 모두 11px로 통일 (단순)
```typescript
페이지 헤더:  12px, 700  (페이지 구분용)
1단 헤더:     11px, 700  ✅ 기준
2단 헤더:     11px, 600  ✅ 통일
데이터 셀:    11px, 400  ✅ 통일
4M 열:        10px, 600  (작은 열이라 약간 작게)
플레이스홀더: 11px, 600  ✅ 통일
```

**장점**:
- 모든 워크시트 영역이 11px로 통일
- 읽기 쉬움
- 단순 명료

**단점**:
- 4M 열이 약간 작아질 수 있음

---

### 제안 B: 10px로 통일 (컴팩트)
```typescript
페이지 헤더:  12px, 700  (페이지 구분용)
1단 헤더:     10px, 700  ✅ 통일
2단 헤더:     10px, 600  ✅ 통일
데이터 셀:    10px, 400  ✅ 통일
4M 열:        10px, 600  ✅ 통일
플레이스홀더: 10px, 600  ✅ 통일
```

**장점**:
- 완전 통일 (4M 포함)
- 화면에 더 많은 내용 표시
- 깔끔함

**단점**:
- 약간 작게 느껴질 수 있음

---

### 제안 C: 혼합형 (현실적)
```typescript
페이지 헤더:  12px, 700  (페이지 구분용)
1단 헤더:     11px, 700  ✅ 기준 유지
2단 헤더:     10px, 600  (서브 헤더는 약간 작게)
데이터 셀:    10px, 400  (읽기 쉬운 크기)
4M 열:        10px, 600  (통일)
플레이스홀더: 10px, 600  (통일)
```

**장점**:
- 계층 구조 명확
- 1단 헤더가 돋보임
- 현재와 유사하여 적응 쉬움

**단점**:
- 완전 통일은 아님
