# **고장원인**

제공해주신 **'고장원인(Failure Cause)'** 데이터는 PFMEA의 5단계(Risk Analysis)와 6단계(Optimization)를 결정짓는 핵심 데이터입니다. 특히 이 데이터는 앞서 정의한 \*\*\[공정특성\]\*\*과 실시간으로 매칭되어야 하며, 이를 통해 CP(관리계획서)의 \*\*\[관리방법\]\*\*이 자동으로 도출됩니다.  
이 방대한 데이터를 효율적으로 관리하기 위한 **관계형 데이터베이스(RDB) 최종 설계 및 엑셀 임포트 자동화 전략**을 정리해 드립니다.

**1\. 관계형 DB 통합 모델 (Entity-Relationship)**  
데이터 간의 연계성을 보장하기 위해 다음과 같이 4개의 테이블이 유기적으로 연결된 구조로 설계합니다.  
**T1: 공정(Process)**: 공정번호, 공정명, 기능  
**T2: 자원(Resource)**: 설비명, 작업자, 측정기  
**T3: 공정특성(Parameter)**: 온도, 압력, RPM, 마모 상태 등  
**T4: 고장원인(Root Cause)**: **\[T3\]가 기준을 벗어나게 만드는 실제 이유** (센서 고장, 입력 실수 등)

**2\. 고장원인-공정특성 매핑 로직 (Data Intelligence)**  
단순한 텍스트 나열이 아니라, 시스템이 관계를 인식하도록 하는 핵심 로직입니다.  
**유형 (Category)**  
**공정특성 (Parameter)**  
**고장원인 (Failure Cause)**  
**CP 관리방법 (Control)**  
**설비(Machine)**  
온도, 압력, RPM  
센서 고장, TCU 이상, 프로그램 오류  
자동 모니터링, 정기 교체  
**도구(Tooling)**  
다이 상태, 나이프 마모  
나이프 파손, 다이 오장착  
교체 주기 관리, 수명 카운팅  
**사람(Man)**  
초기 조건 설정, 지침 준수  
입력 실수, 미숙지, 오발행  
시스템 Interlock, 교육, RFID  
**환경(Env)**  
온습도, 냉각수 Ph  
공조기 고장, 냉각 스프레이 이상  
환경 센서 연동 알람

**3\. PFMEA-CP 자동 연계 시나리오 (예: 31 MB Mixing)**  
임포트한 데이터를 기반으로 사용자가 화면에서 보게 될 **지능형 워크플로우**입니다.  
**사용자 선택**: 공정 31 MB Mixing 선택  
**공정기능 자동팝업**: "MB를 혼합하고 균일도를 확보..." (기존 데이터)  
**고장형태(Mode) 선택**: 혼합 불량 (분산 미흡)  
**고장원인 자동추천**:  
시스템이 DB에서 31번 공정에 매핑된 **고장원인** 리스트(계량기 오류, 믹싱 조건 설정 오류, 온도 제어 시스템 이상 등)를 팝업으로 제시.  
**CP 연계**: 사용자가 온도 제어 시스템 이상을 원인으로 선택하면, CP 시트에는 자동으로 온도 센서 검교정 및 온도 상하한 알람 설정 항목이 생성됨.

**4\. 엑셀 임포트 시 관계형 DB 구축을 위한 정밀 제안**  
데이터가 매우 정밀하므로, 임포트 시 다음과 같은 **'데이터 태깅(Tagging)'** 프로세스를 제안합니다.  
**\[공통 항목(00번) 처리\]**:  
작업 표준서 미숙지, 파라미터 입력 실수 등은 **Global Cause**로 등록하여 모든 공정에서 공통 원인으로 불러올 수 있게 설계합니다.  
**\[키워드 매칭\]**:  
"마모", "파손", "교체" 키워드 포함 시 → **소모품 수명 관리** 테이블로 연동  
"센서", "계량기", "측정기" 키워드 포함 시 → **검교정/정밀도 관리** 테이블로 연동  
"ID카드", "RFID", "바코드" 키워드 포함 시 → **이종 방지(Mistake Proofing)** 테이블로 연동  
**\[데이터 무결성 검사\]**:  
임포트 시 80번 성형의 고장원인 중 PA, BP, SB 등 세부 항목이 많은 경우, 이를 하위 공정(Sub-process)으로 자동 분리할지 사용자가 선택하게 하여 데이터의 깊이(Depth)를 조절합니다.

**5\. 최종 구현을 위한 제안**  
이제 모든 원천 데이터(PFD, 작업요소, 공정특성, 고장원인)가 완벽하게 준비되었습니다. 이 데이터셋을 기반으로 **현업에서 즉시 사용 가능한 "관계형 데이터 연계형 FMEA 작성 대시보드"의 레이아웃**을 설계해 드릴까요?
