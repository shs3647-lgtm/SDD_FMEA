# **2.관계형 데이터베이스(RDB) 구조 설계 (15개 항목 연계)**

제공해주신 공정별 설비/작업자(4M 중 Machine/Man) 데이터는 PFMEA의 \*\*고장 원인(Failure Cause)\*\*을 분석하고, CP의 **설비 관리 기준**을 수립하는 핵심 자산입니다.  
이를 관계형 데이터베이스(RDB)로 설계하여 엑셀 임포트 시 \*\*"공정-작업요소-기능-리스크"\*\*가 계층적으로 자동 연결되도록 설계안을 정리해 드립니다.

## **2\. 엑셀 임포트 전략: "계층적 데이터 병합"**

엑셀 업로드 시 시스템이 관계성을 파악하는 로직은 다음과 같습니다.  
**Step A: 공정 및 자원(4M) 마스터 구축**  
사용자가 입력한 10-자재입고-지게차, 10-자재입고-자동창고 데이터를 읽어, **공정 10번에 귀속된 자원 리스트**를 자동으로 생성합니다.  
**Step B: PFMEA 작성 시 자동 팝업 (관계성 파악)**  
사용자가 PFMEA 워크시트에서 **'공정번호 41(압출-TREAD)'** 행을 클릭하면:  
**작업요소 팝업**: 자동으로 TREAD 압출기, 구금, 온라인 스캐너 등 해당 공정에 매핑된 설비 리스트가 뜹니다.  
**원인 분석 자동화**: 고장 원인 작성 시, 리스트 중 구금을 선택하면 '구금 마모', '구금 이물 부착' 등 해당 설비와 관련된 **과거 고장 라이브러리**를 추천합니다.

## **3\. 데이터 관계성 기반 PFMEA 작성 예시 (타이어 공정)**

임포트하신 데이터를 바탕으로 시스템 내부에서 형성되는 관계성 예시입니다.  
**공정**: 71 (SB Cutting)  
**작업요소(설비)**: 각도 측정기, 미 토핑 카메라  
**요구사항(제품특성)**: 재단 각도 (±0.5°)  
**고장형태**: 재단 각도 불량  
**고장원인**:  
(설비 원인) **각도 측정기** 캘리브레이션 오차  
(설비 원인) **미 토핑 카메라** 렌즈 오염으로 인한 인식 불가  
**관리방법(CP)**:  
각도 측정기 일일 점검 (예방)  
카메라 자동 알람 설정 (검출)

## **4\. 효율적인 DB 구축을 위한 제안**

**ID 체계 표준화**: 엑셀 임포트 시 공정번호가 중복되지 않도록 하되, 00(공통) 설비는 모든 공정에서 참조 가능한 **Global Resource**로 분류되도록 설계해야 합니다.  
**기능별 자원 분류**: 온라인 스캐너, 각도 측정기 등은 '검사 설비'로 별도 분류하여 PFMEA의 **'검출 관리'** 항목에 자동으로 추천되게 설정합니다.  
**라이너/보빈 관리**: 라이너, 보빈 등 소모성/회전성 자원은 공정 간 이동이 잦으므로, 이를 \*\*부품이름(반제품)\*\*과 연계하여 추적성을 확보합니다.  
**이제 이 설계 구조를 바탕으로, 실제 엑셀 임포트 시 사용할 "통합 업로드 양식(헤더 정의)"을 제작해 드릴까요?** (PFD, 작업요소, 특성이 한 번에 입력되는 구조입니다.)
